---
title: "Search By Label"
api: "GET /api/arbitrum/search/{searchtext}"
description: " This endpoint enables you to search and retrieve labels by a given search term. It's particularly useful for finding specific Arbitrum accounts based on labels, types or subtypes."
---

### Inputs

<ParamField path="searchtext" type="string" required>
  Specify the term or part of the search term you wish to search for.
</ParamField>
<ParamField query="apikey" type="string" required>
  Your API key for authentication.
</ParamField>
<ParamField query="limit" type="string">
  Specify number of shown results.
</ParamField>
<ParamField query="offset" type="string">
  Specify the starting point in the list of results for pagination. 
  e.g. `offset=0`, `offset=1`, etc..
</ParamField>

### Response

<ResponseField name="data" type="array">
  An array that encapsulates the labels corresponding to the search term provided.

<Expandable title="Label Object">

<ResponseField name="blockchain" type="string">
  Name of the blockchain network.
</ResponseField>

{" "}

<ResponseField name="address_name" type="string">
  The designated name associated with the specific Arbitrum address.
</ResponseField>

{" "}

<ResponseField name="address" type="string">
  The Arbitrum address for the label.
</ResponseField>

{" "}

<ResponseField name="label_type" type="string">
  Categories like 'token', 'exchange', etc., to define the nature of the label.
</ResponseField>

{" "}

<ResponseField name="label_subtype" type="string">
  Additional attributes, like token_contract_address or exchange_name, for a
  more granular understanding of the label.
</ResponseField>

{" "}

<ResponseField name="label" type="string">
  The general name or title of the label.
</ResponseField>

  </Expandable>
</ResponseField>

```bash

--header 'Content-Type: application/json'

```
